四、设计思路

1，概述
1.1，系统目标
通过增加服务层实现外挂业务进入主流程，保证各个模块独立运行维护
1.2，系统实现原则
通过系统分层完成各个层级业务的分离
通过模块的封装完成业务层方便调用
新框架增加系统服务层
将工作流，外挂体系放在系统最底层
1.3，代码规范
