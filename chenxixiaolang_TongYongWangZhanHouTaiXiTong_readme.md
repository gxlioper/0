
### 背景

考虑到公司业务系统较多，各业务系统后台各自开发维护用户及权限系统浪费开发资源，并且结合之前在公司做项目经验，决定开发一套基于SSM的统一管理后台来简化公司的业务系统

在整理说一下关于后台模板、本人已经获得larry作者的授权 
## 2.0版本预计在下面项目中更新(时间不定)
下面更新大概介绍
基于SD-Admin企业通用后台做的消息服务平台(给其他系统提供统一的消息接入，可以处理未读、已读、消息列表等，消息支持发送邮件)
